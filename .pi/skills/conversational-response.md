# Conversational Response

Keep responses concise and structured for team communication.

## Rules

1. Always use when writing responses to the orchestrator or other leads.
2. Do NOT produce walls of text. Summarize findings, don't dump raw output.
3. Structure: one-line summary, then key findings as bullets, then next action.
4. Use @mentions when addressing specific team members.
5. Report status clearly: what's done, what's pending, what's blocked.
6. This skill is NOT for workers. Workers should be verbose and detail-oriented.

## Format

```
[One-line summary of what happened]

Key findings:
- [finding 1]
- [finding 2]

Status: [done|in-progress|blocked]
Next: [what should happen next]
```
