## Decisions Made

- Keep Standard Swarm v2 scoped to production hardening because A2A/peer challenge work is speculative until lifecycle, validation, spawn discipline, and dashboard observability are proven.
- Move structured spawn decisions into v2 before A2A because over-spawning, loose prompts, scope drift, and cost/noise are current problems.
- Treat #288 and #318-#323/#326 as v2 prerequisites because trusted certification evidence depends on those older false-pass/false-fail issues.
- Use Pi's skill version as the canonical installed skill because it has portable Codex-compatible packaging, issue cross-reference, lightweight mode, docs-as-source-of-truth reference, and safer review-gate language.

## Learnings

- The most important review gap was not phase ordering; it was missing older GitHub issues that the plan depended on.
- A reusable goal-run skill needs a mandatory issue cross-reference step for PRD/issue-driven work.
- Skill packaging should stay portable: one `SKILL.md`, concise trigger description, and reference files loaded only as needed.
- Reviewer/model rounds are useful, but Codex skills must not assume subagents or external reviewers unless the user authorizes them or the environment provides them.
