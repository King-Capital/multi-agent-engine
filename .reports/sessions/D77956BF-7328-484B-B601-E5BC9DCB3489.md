## Decisions Made

1. **`mae expert` added beyond issue spec** — user wanted both `learn` (persistent expertise) and `expert` (instant interactive session). The expert command auto-learns if no cached expertise exists, then drops into REPL. This makes the codebase-understanding workflow seamless.
2. **Raw fetch for CLI LLM calls, not adapter pattern** — new CLI commands call LiteLLM directly via `llm-gateway.ts` instead of going through the adapter. Adapters are for session-managed agent delegation with cost tracking, budget enforcement, and healing. CLI tools are standalone one-shot calls where that overhead is unnecessary. Model routing still used via `resolveModel()`.
3. **Orchestrator split boundary** — orchestrator owns session lifecycle (create, pause, resume, shutdown, steer commands, adapter management). chain-runner owns execution (step dispatch, retry loops, till-done verification, agent delegation). Connected via `ChainRunnerDeps` interface — similar dependency injection pattern to existing `TeamExecutionDeps`.
4. **Fix everything found, not just "our changes"** — user explicitly reinforced the "fix what you find" rule. Pre-existing efficiency issues (orchestrator-loop rebuilding prompt every cycle, regex recompilation in hot paths) were fixed in the same PR rather than deferred.
5. **Full swarm before commit** — used the 5-agent CC review swarm pattern (correctness, adversarial, quality, security, domain specialist) plus full-app scope reviewers. Two rounds total, ~80 findings, all CRITICAL/HIGH/MEDIUM fixed.
6. **`isInternalUrl` in security.ts not expertise-builder.ts** — SSRF check is a security utility, belongs with other security functions for reuse by any future URL-fetching code.

## Learnings

- `...process.env` spread in subprocess env passes EVERYTHING including secrets you think you excluded. Must destructure-and-omit explicitly.
- `@clack/prompts` spinner + interactive prompts don't compose — calling an interactive function inside a spinner corrupts terminal output. The spinner must stop before any nested interactive code runs.
- The 5-agent standard swarm found significantly more issues when scoped to the full app vs. just the diff. Diff-only reviews miss integration bugs, pre-existing issues surfaced by new code paths, and architectural problems.
- `max_expertise_lines` was defined in PersonaConfig but never wired into `buildSystemPrompt` — a silent token waste on every agent call with large expertise files.
